# Contributing

Thanks for your interest in contributing to jsonpathx!

## Code of Conduct

Please be respectful and considerate in all interactions. By participating, you agree to uphold a welcoming and harassment-free environment.

## How to Contribute

### 1. Discuss changes first

For substantial changes, open an issue to discuss your proposal before starting work.

### 2. Fork and branch

- Fork the repository
- Create a feature branch from `main`

```bash
git checkout -b your-feature-name
```

### 3. Develop

- Follow the existing coding style
- Keep changes focused and well-scoped
- Add or update tests when applicable
- Update documentation when behavior changes

### 4. Run checks

```bash
npm install
npm test
npm run bench
npm run docs:build
```

### 5. Commit

- Use clear, concise commit messages
- Keep commits logically grouped

### 6. Open a Pull Request

- Describe the change and motivation
- Link relevant issues
- Include screenshots or logs if helpful

## Reporting Issues

When filing a bug, include:

- A minimal reproduction
- Expected vs. actual behavior
- Environment details (Node.js version, OS)
